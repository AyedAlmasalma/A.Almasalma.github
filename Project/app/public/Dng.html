Digital Negative (DNG)

Basics

The DNG (Digital Negative) RAW file format is a TIFF file format standard developed by Adobe. It is used by various digital cameras to store raw sensor data. The DNG format preserves the unprocessed sensor data, providing higher quality and precision (typically 12 or 14 bits per color) compared to lossy JPEG compression. This high-quality data is essential for post-processing and maintaining image integrity.

Structure

The basic structure of a DNG file contains a TIFF header, a file is defined to be a sequence of 8-bit bytes, where the bytes are numbered from 0 to N. The largest possible TIFF file is 2**32 bytes in length. A TIFF file begins with an 8-byte image file header that points to an image file directory (IFD). An image file directory contains information about the image, as well as pointers to the actual image data.


The DNG format is based on the TIFF/EP (Tagged Image File Format/Electronic Photography) standard, which is an extension of the widely-used TIFF format. So DNG contains a TIFF file begins with an 8-byte image file header, the most important information in these headers can be found in the first 2 bytes in the TIFF header, which either have the value 4949.H or 4D4D.H which respectively mean that the file is using either little endian or a big endian byte order.


The DNG file may have multiple IFDs, each serving a specific purpose. For example, IFD#0 typically contains the primary image data, and subsequent IFDs may contain metadata, thumbnails, and other relevant information.

For a comprehensive explanation of the DNG format and TIFF specification, you can refer to the Adobe DNG Specification and TIFF specification, check out Digital Negative (DNG) Specification and TIFF 6.0 Specification in this project, we will focus on the aspects most relevant to our objectives and the user's understanding.
